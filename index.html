Direct Memory Access (DMA) & Overlap

Optimization relies on preventing the CPU from sitting idle while slow I/O operations occur.

Polling vs Interrupts

Polling: CPU actively checks if device is ready (wastes cycles).

Interrupts: CPU works on other tasks. Device controller sends a signal (Interrupt) when I/O is complete.

DMA Architecture

Bypasses the CPU entirely for data transfer. Data flows directly between the Primary Memory and the Device Controller.

Hardware & Driver Buffering

Buffers act as temporary storage to smooth out speed differences between data producers and consumers.

Single Buffering

Controller reads block i while Process reads block i-1.

Double Buffering

Two buffers (A & B). Swaps back and forth to ensure continuous flow.

Ring Buffer

Circular array of buffers. Best for rapid, continuous data streams.

Module 02

Disk Scheduling Algorithms

Access Time Components:

  • Seek Time: Delay while R/W head moves to destination track. (Dominates access time)
  • Latency Time: Rotational delay waiting for sector to spin under head.
  • Transfer Time: Time to copy bits from disk surface to memory.

Access Time = Seek + Latency + Transfer

Scenario Parameters

Tracks 0 to 99. Head starts at 75.

Queue Requests: 23, 87, 36, 93, 66

FCFS (First-Come, First-Served) 251 steps

Path: 75 → 23 → 87 → 36 → 93 → 66

Math: |75-23| + |23-87| + |87-36| + |36-93| + |93-66|

52 + 64 + 51 + 57 + 27 = 251

SSTF (Shortest Seek Time First) 107 steps

Always go to the closest track next.

Path: 75 → 66 → 87 → 93 → 36 → 23

Math: 9 + 21 + 6 + 57 + 13

Total = 107

SCAN (Elevator Algorithm) 100 steps

Move in one direction all the way to the end (99), then reverse.

Path: 75 → 87 → 93 → 99 → 66 → 36 → 23

Math: 12 + 6 + 6 + 33 + 30 + 13

Total = 100

LOOK 87 steps

Like SCAN, but stops at the last request in that direction (93), doesn't go to 99.

Path: 75 → 87 → 93 → 66 → 36 → 23

Math: 12 + 6 + 27 + 30 + 13

Total = 87

C-SCAN (Circular)

Hits 99, returns to 0 (home), goes up again.

Path: 75 → 87 → 93 → 99 → 23 → 36 → 66

90 + home steps

C-LOOK

Hits 93, returns to lowest request (23), goes up.

Path: 75 → 87 → 93 → 23 → 36 → 66

84 + home steps

3.1 Data Hierarchy

Basic element. Single value (length & type).

Record

Collection of related fields treated as a unit.

File

Collection of similar records. Single entity.

Database

Collection of related data with relationships.

Indexed Sequential

Main file + Index file + Overflow file. Drastically reduces average accesses (e.g., 500k vs 1000).

Direct File

Directly access a block at a known address using a required Key field. Fastest single access.

3.3 Directory Structures

Simple Directory

Single list. No structure. Forces user to ensure completely unique file names globally.

Two-Level Scheme

Master directory points to individual User directories. Resolves basic naming collisions.

Hierarchical / Tree-Structured

Master directory with subdirectories and files as entries. Files located via Pathnames.

3.4 Record Blocking

Fixed Blocking

Fixed record size. Causes internal fragmentation (waste due to fitting records into fixed block sizes).

Variable Blocking: Spanned

Records can span across multiple blocks. No wasted space, but requires complex pointer tracking.

Variable Blocking: Unspanned

Records cannot span blocks. Wastes space if the remaining block space is too small for the next record.

Module 04

Allocation & OS Specifics

Contiguous

Single set of continuous blocks. Only starting block and length in FAT.

Issue: External fragmentation (requires compaction).

Chained

Each block contains a pointer to the next block. No external fragmentation.

Issue: Cannot accommodate principle of locality.

Indexed

FAT contains a separate one-level index for each file (entry for every portion).

Best of both worlds: Allows random access without fragmentation.

UNIX / Linux VFS

  • Inodes: Index node. Control structure with key info for a particular file.
  • Superblock object: Represents specific mounted file system.
  • Dentry object: Represents a specific directory entry.
  • File Types: Regular, Directory, Special, Named pipes, Links.

Windows NTFS

  • Key Features: Recoverability, Security, Large disks, Multiple data streams.
  • Sector: Smallest physical storage unit on disk.
  • Cluster: One or more contiguous sectors.
  • Volume: Logical partition on a disk.
